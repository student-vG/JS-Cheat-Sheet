1.Variables and Data Types

Variables:


                var a;                          // variable
                var b = "init";                 // string
                var c = "Hi" + " " + "Joe";     // = "Hi Joe"
                var d = 1 + 2 + "3";            // = "33"
                var e = [2,3,5,8];              // array
                var f = false;                  // boolean
                var g = /()/;                   // RegEx
                var h = function(){};           // function object
                const PI = 3.14;                // constant
                var a = 1, b = 2, c = a + b;    // one line
                let z = 'zzz';                  // block scope local variable

  • var: Old method, not recommended.
  • let: Modern, for changeable values.
  • const: For values that should not change.

    • Data Types:

    
                let name = "John";         // String
                let age = 30;              // Number
                let isHappy = true;        // Boolean
                let fruits = ["Apple", "Banana"]; // Array
                let person = { name: "John", age: 30 }; // Object
                let empty = null;          // Null
                let notAssigned;           // Undefined

    Type Conversion :

    
                let strNum = String(123);  // "123" (Number to String)
                let numStr = Number("42"); // 42 (String to Number)

    2. Operators

    Arithmetic Operators:

    
                let sum = 5 + 3;    // 8 (Addition)

                let prod = 5 * 3;   // 15 (Multiplication)

                let andCondition = (true && false); // false

                let notCondition = !true;            // false

    3. Strings

    7. Loops and Control Flow

    If Statement:

    
                let age = 18;
                if (age >= 18) {
                    console.log("You are an adult.");
                }

    If-Else Statement:

    
                let age = 16;
                if (age >= 18) {
                    console.log("You are an adult.");
                } else {
                    console.log("You are a minor.");
                }

    Nested If Statement:

    
                let age = 20;
                if (age >= 18) {
                    console.log("You are an adult.");
                    if (age >= 65) {
                        console.log("You are a senior citizen.");
                    }
                }

    For Loop:

    
                for (let i = 0; i < 5; i++) {
                    console.log("Iteration: " + i);
                }

    While Loop:

    
                let i = 0;
                while (i < 5) {
                    console.log("Iteration: " + i);
                    i++;
                }

    Do-While Loop:

    
                let i = 0;
                do {
                    console.log("Iteration: " + i);
                    i++;
                } while (i < 5);

    8.DOM (Document Object Model)

    What is the DOM?

    The DOM is a programming interface for web documents. It represents an HTML or XML document as a tree of objects. Each element (like <div>, <h1>, etc.) in an HTML document becomes a node in the DOM tree. JavaScript can interact with and manipulate the DOM to dynamically change the content, structure, or style of a web page.

    Accessing DOM Elements:

    1. Select Elements by ID:

    
                let myDiv = document.getElementById("myDiv");
                console.log(myDiv.innerHTML); // Outputs: Hello, World!

    2. Select Elements by Class:

    
                let boxes = document.getElementsByClassName("box");
                console.log(boxes[0].innerHTML); // Outputs: Box 1

    3. Select Elements by Tag Name:

    
                let paragraphs = document.getElementsByTagName("p");
                console.log(paragraphs[1].innerHTML); // Outputs: Paragraph 2

    4. Query Selectors:

    
                let firstBox = document.querySelector(".box"); // Selects the first element with class 'box'
                console.log(firstBox.id); // Outputs: firstBox
    
                let allBoxes = document.querySelectorAll(".box"); // Selects all elements with class 'box'
                console.log(allBoxes.length); // Outputs: 2

    Manipulating DOM Elements:

    1. Change Content:

    
                document.getElementById("myDiv").innerHTML = "Updated Text!";

    2. Change Style:

    
                document.getElementById("myDiv").style.color = "blue";
                document.getElementById("myDiv").style.fontSize = "20px";

    3. Add/Remove Classes:

    
                let div = document.getElementById("myDiv");
                div.classList.remove("red");
                div.classList.add("blue"); // Changes text color to blue

    4. Create and Append Elements:

    
                let newItem = document.createElement("li");
                newItem.innerHTML = "Item 2";
                document.getElementById("list").appendChild(newItem);

    5. Remove Elements:

    
                let list = document.getElementById("list");
                let item2 = document.getElementById("item2");
                list.removeChild(item2); // Removes 'Item 2'

    Event Handling:

    1. Event Listener:

    
                let button = document.getElementById("myButton");
                button.addEventListener("click", function () {
                    alert("Button clicked!");
                });

    2. Inline Event Handling:

    
                button onclick="alert('Inline click!')">Click Me button

    3. Remove Event Listener:

    
                let button = document.getElementById("myButton");
                function handleClick() {
                    alert("Button clicked!");
                }
                button.addEventListener("click", handleClick);
                button.removeEventListener("click", handleClick); // Removes the event

    4. Common Event Types:

    • click: Triggered when an element is clicked.
    • mouseover: Triggered when the mouse hovers over an element.
    • mouseout: Triggered when the mouse leaves an element.
    • keydown: Triggered when a key is pressed.
    • keyup: Triggered when a key is released.
    • change: Triggered when an input value changes.
    • submit: Triggered when a form is submitted.

    Event Object:

    
                

                let button = document.getElementById("myButton");
                button.addEventListener("click", function (event) {
                    console.log(event.type);  // Outputs: click
                    console.log(event.target); // Outputs: 
                });

    Prevent Default Behavior:

    
                document.getElementById("myLink").addEventListener("click", function (event) {
                    event.preventDefault();
                    alert("Link click prevented!");
                });

    Event Delegation:

    
                let list = document.getElementById("list");
                list.addEventListener("click", function (event) {
                    if (event.target.tagName === "LI") {
                        alert(event.target.innerHTML); // Alerts clicked list item
                    }
                });

    10. Asynchronous JavaScript

    Promises:

    A Promise represents a value that may be available now, or in the future, or never. It's a way of handling asynchronous operations like data fetching.

    
                let promise = new Promise((resolve, reject) => {
                    let success = true;
                    if(success) {
                        resolve("Operation Successful");
                    } else {
                        reject("Operation Failed");
                    }
                });
                
                promise
                    .then(result => console.log(result)) // "Operation Successful"
                    .catch(error => console.log(error)); // "Operation Failed"

    Async/Await:

    The async/await syntax makes working with asynchronous code more readable. It is based on Promises.

    
                async function fetchData() {
                    try {
                        let response = await fetch('https://jsonplaceholder.typicode.com/posts');
                        let data = await response.json();
                        console.log(data);
                    } catch (error) {
                        console.error('Error fetching data:', error);
                    }
                }
        
                fetchData();

    11. Closures in JavaScript

    What is a Closure?

    A closure is a function that "remembers" its lexical environment, even when the function is executed outside that environment. This is possible because inner functions have access to variables of outer functions.

    
                function outerFunction(outerVariable) {
                    return function innerFunction(innerVariable) {
                        console.log("Outer Variable: " + outerVariable);
                        console.log("Inner Variable: " + innerVariable);
                    };
                }
        
                const closureExample = outerFunction('Outer');
                closureExample('Inner'); // Outer Variable: Outer, Inner Variable: Inner

    9. JavaScript Modules

    What are Modules?

    Modules are a way of organizing code into separate files. This allows for better code structure, maintenance, and reuse. Modules can be imported and exported in JavaScript to share functionality between files.

    Exporting and Importing Modules:

    
        // math.js (Module)
        export function add(a, b) {
            return a + b;
        }
        
        export function subtract(a, b) {
            return a - b;
        }
        
        // app.js
        import { add, subtract } from './math.js';
        
        console.log(add(2, 3)); // 5
        console.log(subtract(5, 2)); // 3

    Default Exports:

    
        // myModule.js
        export default function greet(name) {
            return "Hello, " + name;
        }
        
        // app.js
        import greet from './myModule.js';
        console.log(greet('Alice')); // Hello, Alice

    12. JavaScript Design Patterns

    Module Pattern:

    The Module Pattern is used to create private and public methods in an object to maintain encapsulation.

    
                const CounterModule = (function() {
                    let count = 0;
        
                    return {
                        increment: function() {
                            count++;
                            console.log(count);
                        },
                        decrement: function() {
                            count--;
                            console.log(count);
                        },
                        getCount: function() {
                            return count;
                        }
                    };
                })();
        
                CounterModule.increment(); // 1
                CounterModule.increment(); // 2
                CounterModule.decrement(); // 1
                console.log(CounterModule.getCount()); // 1

    Singleton Pattern:

    The Singleton Pattern ensures that a class has only one instance, and provides a global point of access to it.

    
                const Singleton = (function() {
                    let instance;
        
                    function createInstance() {
                        return { name: "Singleton" };
                    }
        
                    return {
                        getInstance: function() {
                            if (!instance) {
                                instance = createInstance();
                            }
                            return instance;
                        }
                    };
                })();
        
                const instance1 = Singleton.getInstance();
                const instance2 = Singleton.getInstance();
        
                console.log(instance1 === instance2); // true

    13. Higher-Order Functions

    What is a Higher-Order Function?

    A higher-order function is a function that either accepts a function as an argument or returns a function. JavaScript has many built-in higher-order functions like map, filter, and reduce.

    Example of a Higher-Order Function:

    
                function multiply(a, b) {
                    return a * b;
                }
        
                function calculate(a, b, operation) {
                    return operation(a, b);
                }
        
                console.log(calculate(5, 3, multiply)); // 15

    Using Built-in Higher-Order Functions:

    
                const numbers = [1, 2, 3, 4, 5];
                
                // map() example
                const doubled = numbers.map(num => num * 2);
                console.log(doubled); // [2, 4, 6, 8, 10]
        
                // filter() example
                const evenNumbers = numbers.filter(num => num % 2 === 0);
                console.log(evenNumbers); // [2, 4]
        
                // reduce() example
                const sum = numbers.reduce((acc, num) => acc + num, 0);
                console.log(sum); // 15

    14. JavaScript Event Loop and Call Stack

    Call Stack:

    The Call Stack is a data structure that keeps track of the execution context. When a function is called, it is added to the call stack, and when it finishes, it is removed.

    
                function first() {
                    console.log("First function");
                }
        
                function second() {
                    console.log("Second function");
                    first();
                }
        
                second(); // "Second function" -> "First function"

    Event Loop:

    The Event Loop checks if the call stack is empty, and if so, it executes any messages in the message queue (such as callbacks or event handlers).

    
                console.log("Start");
        
                setTimeout(function() {
                    console.log("Timeout finished");
                }, 0);
        
                console.log("End"); // "Start" -> "End" -> "Timeout finished"
